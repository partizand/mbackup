AutoSave version 0.1 help

Программа для копирования/синхронизации директорий

Общие сведения
Программа AutoSave, для ОС Win32, позволяет автоматизировать процесс резервного копирования директорий. Возможно копирование, синхронизирование, зеркалирование директорий. AutoSave не требует инсталляции, не изменяет реестр и системные файлы на компьютере.

Лицензионное соглашение
Программа распространяется БЕСПЛАТНО (Freeware) по принципу "как есть". Вы можете использовать программу в личных и коммерческих целях бесплатно. Можете создавать любое количество копий программы и распространять их без каких-либо ограничений. За исключением получения прибыли от распространения.

Описание операций

Действие
Операции
Описание
Копирование Содержимое источника копируется в приемник. Если у одинаковых файлов разные даты, то копирование происходит если файл в приемнике имеет более раннюю дату. Несовпадающие с источником файлы в приемнике остаются как есть.. Простое копирование
Зеркалирование Содержимое источника копируется в приемник. Если у одинаковых файлов разные даты, то копирование происходит если файл в приемнике имеет более раннюю дату. Несовпадающие с источником файлы и директории в приемнике удаляются. Файлы в источнике остаются как есть. Копирование с удалением лишних файлов и директорий в приемнике
Синхронизация Синхронизация файлов в приемнике и источнике. Отсутствующие файлы копируются из источника в приемник и наоборот. Синхронизация каталогов

Настройки программы
В окне настроек программы настраиваются параметры протоколирования (лог файл): имя лог файла, ограничение размера ("0" - без ограничений), кнопка "открыть" позволяет открыть лог файл во внешней программе просмотра. Если задано ограничение лог файла, при превышении размера файла заданного ограничения, лог файл переименовывается в "autosave.bak" в тойже директории и начинает вестись заново. Это сделано, что бы не происходило постоянной перезаписи файла при достижении лимита размера.

Параметры командной строки

В основном используются для консольной версии autosavec.exe, но параметры одинаковы для обоих версий программы. Порядок расположения параметров значения не имеет.

Пример использования:
autosavec -r [-p имя_профайла] [-log имя_логфайла] [-q]
или
autosavec <действие> -source директория_источник -dest директория_получатель [-recurse] [-log имя_логфайла] [-q]

где

-r
При запуске программы запустить все разрешенные задания профиля
Задания выполняются по очереди сверху вниз. Для регулирования очередности используйте кнопки вверх/вниз

 -p имя_профайла
Загрузить указанный профайл. Если файл находится в директории с программой, достаточно указать только имя файла без полного пути. Настройки загрузки профилей программы в этом случае игнорируются. Может быть указано только одно имя профайла.  Если не укзано используется значение из файла autosave.ini. Если указано, то заданные в командной строке <действие>, источник, приемник игнорируются.

<Действие>
copy
Аналогично операции "Копирование", см. таблицу выше
mirr
Аналогично операции "Зеркалирование", см. таблицу выше
sync
Аналогично операции "Синхронизация", см. таблицу выше

-source директория_источник
Каталог источник

-dest директория_получатель
Каталог получатель

-recurse
Обрабатывать все подкаталоги

-log имя_логфайла
Указание файла для записи логов. Если не указан, параметр берется из настроек autosave.ini.

-q
Выйти из программы по завершении всех заданий (используется вместе с параметром -r) . Имеет смысл только в графической версии


Дополнительная информация
В катлоге с программой могут находятся файлы:
default.xml - файл содержащий сведения о задачах - профайл
autosave.ini - файл с настройками программы
autosave.log - лог файл (возможно задать любое другое имя лог файла через настройки программы)
english.lng - файл перевода на английский язык
Но для работы достаточно EXE-шника

Для выполнения задачи копирования/синхронизации запускается отдельный поток. Если есть одновременно несколько задач на выполнение, то они ставятся в очередь и выполняются по одной. Это сделано для возможности задавать на одно время неограниченного числа заданий не загружая при этом компьютер.

Сама программа не содержит встроенного планировщика. Выполнение заданий происходит при запуске программы с параметром -r . Или при запуске вручную в самой программе. Не считаю нужным повторять планировщик windows.

Перевод
Если хотите перевести программу на другой язык:
  1. Переименуйте english.lng, с сохранением расширения файла.
  2. В новом файле, в секции [Language] укажите название языка и файл помощи
  3. Переведите строки
  4. Скопируйте новый файл в каталог с программой
В настройках программы появится новый язык
